# firmware

RP2350-Zero sketch: scans the panel's buttons and pots, emits USB MIDI per
[docs/midi-contract.md](../docs/midi-contract.md). The control tables live in
[chaossynth/mapping.h](chaossynth/mapping.h), generated by layout-tool from
synth/mapping.json — a wiring or mapping change never means editing the sketch.

## Build

    brew install arduino-cli   # once
    ./build.sh                 # compile (also runs the pot-filter host check)
    ./build.sh --flash         # compile + upload (board attached via USB)

The core and libraries are version-pinned and installed under `firmware/.arduino/`
(gitignored), so nothing touches a global Arduino setup and rebuilds work offline
once primed. Arduino IDE users: the setup lines are in the sketch header comment.

If `--flash` can't find the board, replug the USB cable while holding BOOTSEL and
run it again.

## Behavior notes

- The firmware idles until a USB host enumerates it. On every mount — first boot
  or a cable re-plug — it sends CC 123 value 0 before anything else.
- Expanders self-heal: probed every 250 ms, configured when they (re)appear,
  pullups re-asserted on healthy chips because a brownout-reset MCP23017 still
  ACKs but has lost them. The serial console (115200 baud) logs
  `expander N up` / `expander N LOST`.
- Buttons debounce at 10 ms. Pots go through EMA + deadband
  ([chaossynth/pot_filter.h](chaossynth/pot_filter.h)); the invariant that a
  resting pot is silent but a sweep still reaches 0 and 127 is checked on the
  host by [test/pot_filter_test.cpp](test/pot_filter_test.cpp) on every build.

## Hardware checklist (Nevyn)

Flash, then `brew install receivemidi` and watch `receivemidi dev Chaossynth`:

1. [ ] Replug the panel: CC 123 value 0 appears once, before any other message.
2. [ ] Every button: exactly one NoteOn (note = its id 0-17, velocity 127) on
       press and one NoteOff on release. Mash aggressively — no doubles.
3. [ ] Every pot: a full sweep reaches both 0 and 127 and moves cleanly.
4. [ ] Hands off everything for two minutes: zero MIDI traffic.
5. [ ] Pull the expander's I2C wiring mid-run: its buttons go quiet and any held
       notes release within ~250 ms (a brief garbage burst right at the yank is
       possible); replug and they work again within ~250 ms, no reboot needed.
