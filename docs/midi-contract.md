# MIDI + mapping contract (v1)

The frozen interface between firmware, layout tool, and synth. All three are built
in parallel against THIS file. Changes here require Nevyn's sign-off and updating
every consumer — bump `version` when the mapping schema breaks.

## Identity

- The panel enumerates as a USB MIDI device named **"Chaossynth"** (TinyUSB string
  descriptor — already set in the firmware).
- The synth selects its MIDI input by that name. Test harnesses (virtual panels)
  MUST name their virtual port "Chaossynth" too, so the synth can't tell the
  difference.
- Everything is on **MIDI channel 1**.

## Controls

Every physical control has a stable numeric **id**, assigned by the layout tool.
Ids are never renumbered; a new control takes the next free id for its kind.
Buttons and pots have separate id spaces (button 0 and pot 0 coexist). The id IS
the MIDI assignment:

| Event | MIDI out |
|---|---|
| Button `id` pressed | NoteOn(note = id, vel = 127) |
| Button `id` released | NoteOff(note = id, vel = 0) |
| Pot `id` moved | CC(20 + id, value 0–127) |

Button ids: 0–31. Pot ids: 0–7.

Special buttons — the escape hatch for controls that must emit something other
than a note (default is always `note`):

- `cc_momentary`: CC(cc, 127) on press, CC(cc, 0) on release
- `cc_toggle`: CC(cc, 127) / CC(cc, 0) alternating per press
- Special-button CCs are assigned from **40 upward** — never colliding with pots.

Reserved / unused: notes 32–127, CC 0–19, CC 28–39, CC 60–119, pitch bend,
aftertouch. Don't emit them, don't listen for them.

## Lifecycle

- Firmware sends **CC 123 (All Notes Off), value 0** on every USB mount —
  first boot and any cable re-plug — before anything else. (Amended 07-14:
  was "once at boot"; every-mount is what ships, and it's safer after yanks.)
- On mount, each pot also emits one baseline CC so the synth starts from the
  knobs' physical reality instead of stale values.
- Synth, on CC 123: release every held voice.
- Synth must **log and ignore** any note/CC not in the mapping — never crash,
  never guess. Unmapped input is a mapping bug to surface, not an error to hide.
- Synth tolerates the MIDI source appearing late (panel may enumerate after the
  synth starts) and re-appearing after a cable yank.

## mapping.json

Canonical path: `synth/mapping.json`. Written by the layout tool, read by the
synth, and the source of truth from which the firmware header is generated.

```json
{
  "version": 1,
  "image": "layout-tool/panel.jpg",
  "hardware": {
    "i2c": { "sda": 4, "scl": 5 },
    "expanders": [32],
    "muxes": [{ "s0": 6, "s1": 7, "s2": 8, "adc": 26 }]
  },
  "controls": [
    {
      "id": 0,
      "kind": "button",
      "label": "big red mushroom",
      "pos": { "x": 0.42, "y": 0.13 },
      "wiring": { "source": "expander", "chip": 0, "pin": 3 },
      "midi": { "type": "note", "note": 0 },
      "synth": {}
    },
    {
      "id": 0,
      "kind": "pot",
      "label": "wobble amount",
      "pos": { "x": 0.7, "y": 0.5 },
      "wiring": { "source": "mux", "mux": 0, "channel": 4 },
      "midi": { "type": "cc", "cc": 20 },
      "synth": {}
    }
  ]
}
```

- `pos` is normalized 0–1 on the panel photo, `null` until placed.
- `wiring.source` is one of:
  - `"pico"` — direct GPIO button: `{ "source": "pico", "gpio": 2 }`
  - `"expander"` — MCP23017: `{ "source": "expander", "chip": 0, "pin": 3 }`
    (`chip` indexes `hardware.expanders`, `pin` 0–15 = GPA0..GPB7)
  - `"pico_adc"` — direct ADC pot: `{ "source": "pico_adc", "gpio": 27 }` (26–29)
  - `"mux"` — 4051 channel: `{ "source": "mux", "mux": 0, "channel": 4 }`
- `midi.type` is `"note"` (buttons, default), `"cc"` (pots), `"cc_momentary"` or
  `"cc_toggle"` (special buttons).
- `synth` is a free-form object owned by patch design — schema now pinned in
  docs/plans/patch-design.md. The synth ignores fields it doesn't know; the
  layout tool round-trips it untouched.
- Consumers must refuse a file whose `version` they don't know.

## mapping.h

Generated by the layout tool into `firmware/chaossynth/mapping.h` from the same
data: the `expanderAddresses[]`, `muxes[]`, `buttons[]`, `pots[]` tables and I2C
pin constants (the table shapes are the ones in chaossynth.ino — the firmware plan
moves them into this header). Rules:

- First line: `// GENERATED by layout-tool — do not hand-edit.`
- No timestamps in the output (they make noisy diffs); content only.
- A checked-in copy always exists, so firmware builds without running the tool.
